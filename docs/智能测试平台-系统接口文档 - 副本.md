## 全局说明

### 响应通用结构

大多数接口返回统一的响应格式：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

### 认证说明

- 登录接口返回 `authorization` 字段，格式为 `Basic Base64(username:password)`
- 需要认证的接口需在请求头中添加 `Authorization` 字段

```
Authorization: Basic eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 分页参数

部分列表接口支持分页查询：

| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| page | Integer | 否 | 页码 | 1 |
| size | Integer | 否 | 每页条数 | 10 |

## 一、认证模块 (Auth)

**基础路径**: `/api/auth`

### 1.1 用户登录

**接口说明**: 用户登录获取认证令牌

| 项目 | 说明 |
|------|------|
| 接口名称 | 用户登录 |
| 请求方式 | POST |
| 请求路径 | `/api/auth/login` |
| Content-Type | `application/json` |
| 是否需要认证 | 否 |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户账号 |
| password | String | 是 | 用户密码 |

#### 请求示例

```json
{
  "username": "admin",
  "password": "123456"
}
```

#### 成功响应

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 登录是否成功 |
| message | String | 响应消息 |
| authorization | String | Basic认证令牌 |
| username | String | 用户名 |

#### 成功响应示例

```json
{
  "success": true,
  "message": "Login successful.",
  "authorization": "Basic YWRtaW46MTIzNDU2",
  "username": "admin"
}
```

#### 失败响应示例

```json
{
  "success": false,
  "message": "Invalid username or password."
}
```

### 2.4 创建项目

**接口说明**: 创建新项目

| 项目 | 说明 |
|------|------|
| 请求方式 | POST |
| 请求路径 | `/api/projects` |
| Content-Type | `application/json` |
| 是否需要认证 | 是 |

#### 请求体

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | String | 是 | 项目名称 |
| description | String | 否 | 项目描述 |
| owner | String | 否 | 项目负责人 |

#### 请求示例

```json
{
  "name": "新项目",
  "description": "项目描述",
  "owner": "admin"
}
```

#### 响应参数

返回创建的 `Project` 对象。
