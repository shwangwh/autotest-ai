# 登录模块接口文档

## 1. 接口概述

| 项目 | 说明 |
|------|------|
| 接口名称 | 用户登录 |
| 请求方式 | POST |
| 请求路径 | `/api/auth/login` |
| Content-Type | application/json |

---

## 2. 请求参数

### 2.1 请求体参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户账号 |
| password | String | 是 | 用户密码 |

### 2.2 请求示例

```json
{
  "username": "admin",
  "password": "123456"
}
```

---

## 3. 响应参数

### 3.1 成功响应

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 登录是否成功 |
| message | String | 响应消息 |
| token | String | 认证令牌 |

### 3.2 成功响应示例

```json
{
  "success": true,
  "message": "登录成功",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### 3.3 失败响应示例

```json
{
  "success": false,
  "message": "账号或密码错误"
}
```

---

## 4. 错误码说明

| 错误码 | 说明 |
|--------|------|
| success: true | 登录成功 |
| success: false | 登录失败，账号或密码错误 |

---

## 5. 前端调用示例

### 5.1 Vue.js 调用示例

```javascript
async login() {
  try {
    const response = await fetch('/api/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        username: this.form.username,
        password: this.form.password
      })
    });

    const data = await response.json();

    if (data.success) {
      // 保存token到localStorage
      localStorage.setItem('token', data.token);
      // 跳转到项目列表页
      this.$router.push('/projects');
    } else {
      this.error = data.message;
    }
  } catch (error) {
    this.error = '网络错误，请稍后重试';
  }
}
```

### 5.2 Axios 调用示例

```javascript
import axios from 'axios';

async login() {
  try {
    const response = await axios.post('/api/auth/login', {
      username: this.form.username,
      password: this.form.password
    });

    if (response.data.success) {
      localStorage.setItem('token', response.data.token);
      this.$router.push('/projects');
    } else {
      this.error = response.data.message;
    }
  } catch (error) {
    this.error = '网络错误，请稍后重试';
  }
}
```

## 6. 安全建议

1. **密码传输**：生产环境中应使用 HTTPS 加密传输
2. **密码存储**：后端应使用 BCrypt 或 Argon2 等强哈希算法存储密码
3. **Token 安全**：
   - Token 应设置合理的过期时间
   - 敏感操作应要求重新验证 Token
   - 退出登录时应使 Token 失效
4. **防止暴力破解**：
   - 建议增加登录失败次数限制
   - 多次失败后添加图形验证码或短信验证
   - IP 频率限制

## 7. 后续扩展接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用户登出 | POST | `/api/auth/logout` | 退出登录，使 Token 失效 |
| 刷新 Token | POST | `/api/auth/refresh` | 刷新认证令牌 |
| 注册用户 | POST | `/api/auth/register` | 用户注册 |
| 修改密码 | PUT | `/api/auth/password` | 修改密码 |
