## 全局说明

### 响应通用结构

大多数接口返回统一的响应格式：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

### 认证说明

- 登录接口返回 `authorization` 字段，格式为 `Basic Base64(username:password)`
- 需要认证的接口需在请求头中添加 `Authorization` 字段

```
Authorization: Basic eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 分页参数

部分列表接口支持分页查询：

| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| page | Integer | 否 | 页码 | 1 |
| size | Integer | 否 | 每页条数 | 10 |

## 一、认证模块 (Auth)

**基础路径**: `/api/auth`

### 1.1 用户登录

**接口说明**: 用户登录获取认证令牌

| 项目 | 说明 |
|------|------|
| 接口名称 | 用户登录 |
| 请求方式 | POST |
| 请求路径 | `/api/auth/login` |
| Content-Type | `application/json` |
| 是否需要认证 | 否 |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户账号 |
| password | String | 是 | 用户密码 |

#### 请求示例

```json
{
  "username": "admin",
  "password": "123456"
}
```

#### 成功响应

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 登录是否成功 |
| message | String | 响应消息 |
| authorization | String | Basic认证令牌 |
| username | String | 用户名 |

#### 成功响应示例

```json
{
  "success": true,
  "message": "Login successful.",
  "authorization": "Basic YWRtaW46MTIzNDU2",
  "username": "admin"
}
```

#### 失败响应示例

```json
{
  "success": false,
  "message": "Invalid username or password."
}
```

## 八、前端调用示例

### 8.1 登录调用

```javascript
async login() {
  try {
    const response = await fetch('/api/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        username: this.form.username,
        password: this.form.password
      })
    });

    const data = await response.json();

    if (data.success) {
      localStorage.setItem('authorization', data.authorization);
      localStorage.setItem('username', data.username);
      this.$router.push('/projects');
    } else {
      this.error = data.message;
    }
  } catch (error) {
    this.error = '网络错误，请稍后重试';
  }
}
```

### 8.2 带认证的项目列表请求

```javascript
async getProjects() {
  try {
    const authorization = localStorage.getItem('authorization');
    const response = await fetch('/api/projects', {
      headers: {
        'Authorization': authorization
      }
    });
    this.projects = await response.json();
  } catch (error) {
    console.error('获取项目列表失败', error);
  }
}
```

### 8.3 Axios 全局配置

```javascript
import axios from 'axios';

// 设置全局baseURL和认证头
axios.defaults.baseURL = '/api';
axios.interceptors.request.use(config => {
  const authorization = localStorage.getItem('authorization');
  if (authorization) {
    config.headers.Authorization = authorization;
  }
  return config;
});
```

## 九、安全建议

1. **传输安全**: 生产环境中使用 HTTPS 加密传输
2. **密码存储**: 后端使用 BCrypt 等强哈希算法存储密码
3. **认证令牌**: 
   - Basic Auth 令牌设置合理的过期时间
   - 退出登录时使令牌失效
   - 敏感操作重新验证令牌
4. **防暴力破解**:
   - 限制登录失败次数
   - 失败后添加验证码
   - IP 频率限制
5. **权限控制**: 根据用户角色控制接口访问权限
6. **数据验证**: 对所有输入参数进行校验和过滤
