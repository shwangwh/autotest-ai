# 测试点模块优化方案

## 问题分析

1. **测试点列表显示问题**：
   - 测试点列表中"来源需求"显示为"需求 #null"，说明需求名称没有正确显示
   - 提取的测试点可能没有在测试点列表中正确显示

2. **功能缺失**：
   - 测试点页面缺少"新增测试点"按钮
   - 缺少新增测试点的页面和功能

## 解决方案

### 1. 前端修改

#### 1.1 修复需求名称显示问题

**问题原因**：
- 前端组件中`getRequirementName`方法可能没有正确处理`requirementId`为null的情况
- 后端返回的测试点数据中`requirementId`可能为null

**解决方案**：
- 优化`getRequirementName`方法，确保正确处理`requirementId`为null的情况
- 确保后端返回的测试点数据中包含正确的`requirementId`

#### 1.2 添加"新增测试点"按钮

**实现位置**：
- `ProjectWorkbench.vue` 文件中的测试点面板头部

**具体实现**：
- 在测试点面板的`panel-head-right`部分添加"新增测试点"按钮
- 点击按钮后打开新增测试点模态框

#### 1.3 设计新增测试点页面

**模态框设计**：
- 基于提取测试点产生的数据结构，设计一个包含以下字段的表单：
  - 测试点名称（必填）
  - 所属需求（下拉选择）
  - 场景类型（下拉选择：功能测试、性能测试、安全测试、兼容性测试）
  - 风险等级（下拉选择：高、中、低）
  - 自动化建议（下拉选择：建议自动化、人工执行）
  - 描述（文本域）

**实现逻辑**：
- 点击"新增测试点"按钮，打开模态框
- 填写表单数据
- 点击"保存"按钮，调用后端API新增测试点
- 保存成功后，刷新测试点列表

### 2. 后端修改

#### 2.1 添加新增测试点的API接口

**实现位置**：
- `TestPointController.java` 文件

**API设计**：
- **URL**: `/api/test-points`
- **方法**: POST
- **请求体**: 测试点对象（包含name、requirementId、sceneType、riskLevel、automationSuggested、description等字段）
- **响应**: 新增的测试点对象

#### 2.2 实现新增测试点的逻辑

**实现位置**：
- `TestPointService.java` 文件

**核心逻辑**：
- 生成测试点编码（如TP-{projectId}-{sequence}）
- 设置默认值（如createdAt、createdBy）
- 保存测试点到数据库

    this.error = '';

    try {

        ...this.testPointForm,
        projectId: this.$route.params.id
      });
      this.info = '测试点添加成功';

      await this.loadProjectData();
    } catch (error) {
      this.error = error.message || '添加测试点失败';
    } finally {
      this.savingTestPoint = false;
    }
  },
  getRequirementName(requirementId) {
    if (!requirementId) return '无';
    const requirement = this.requirements.find(r => r.id === requirementId);
    return requirement ? requirement.name : `需求 ${requirementId}`;

```java
@Service
public class TestPointService {
    private final TestPointRepository testPointRepository;

    public TestPointService(TestPointRepository testPointRepository) {
        this.testPointRepository = testPointRepository;
    }

    // ... 现有方法

    public TestPoint createTestPoint(TestPoint testPoint) {
        // 生成测试点编码
        String pointCode = generatePointCode(testPoint.getProjectId());
        testPoint.setPointCode(pointCode);
        testPoint.setPointName(testPoint.getName());
        testPoint.setPointType("FUNCTIONAL");
        testPoint.setFunction("测试");
        testPoint.setBusinessRule("Default rule created by user");
        testPoint.setCreatedAt(LocalDateTime.now());
        testPoint.setCreatedBy("user");

        return testPointRepository.save(testPoint);
    }

    private String generatePointCode(Long projectId) {
        // 生成测试点编码，格式为TP-{projectId}-{sequence}
        long count = testPointRepository.countByProjectId(projectId) + 1;
        return String.format("TP-%d-%d", projectId, count);
    }
}
```
